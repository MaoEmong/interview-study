요즘 공고는
어떤 기술을 원할까?

전국 신입·주니어 공고를 모아서, 자주 나오는 필수 기술과 우대사항을 정리했어요.

00 시작 전에

다 갖춰야
지원할 수 있는 건 아니에요

공고의 모든 줄을 채워야 합격하는 게 아니에요. 회사도 신입이 다 갖췄을 거라 기대하지 않아요. "지금 우리가 어디쯤인지" 확인하는 용도로 봐요.

01 요즘 원하는 기술 필수 · 기본기

언어 · 프레임워크

BACK / APP

  • 백엔드 Java + Spring / Spring BootJPA 이해까지 자주 언급
  • 모바일 Flutter / React Native

FRONT

  • JavaScript (ES6+) + ReactVue도 자주, HTML5·CSS3는 기본
  • SSR / SPA, 상태관리

하나하나 완벽할 필요 없어요. 이름이 익숙하고 설명할 수 있으면 출발선은 통과.

// 우대사항은 전부 채우는 게 아니라, 내 강점이 될 한두 개를 고르는 칸

03 핵심

공고는 "이상적인 후보"를
적어둔 거예요

그 모든 걸 갖춘 신입은 거의 없고, 회사도 그걸 알아요. 필수에서 기본기를 보여주고, 우대사항 중 한두 개에 내 경험을 얹으면 충분히 경쟁력 있어요.

03 그런데

필수 요건이 잔뜩인 공고는?

그런 곳도 보통은 전부를 기대하기보다 핵심 2~3개를 봐요. "이 중 하나 이상" 식으로 적힌 경우도 많고요. 정말 모든 걸 빡빡하게 요구하는 곳은 신입에겐 난이도가 높은 자리니, 거기 하나에 매이지 말고 통과 가능한 공고부터 노리면 돼요.

모든 공고가 내 타겟일 필요는 없어요.

04 그래서 우리는

빈 칸을 같이 채우는 게
이 스터디예요

기본기는 전원이 다지고, 우대사항은 각자 강점 될 걸 골라서 파면 돼요. 빈 칸이 보여도 괜찮아요.
